Hi! I'm Maria Cecilia, a specialist in technical documentation and information design.

---

## Problems I solve:

1. **Onboarding**: I create documentation which makes a user's first experience smooth and frictionless. 

2. **Knowledge Management**: I turn messy internal notes and PDFs into a single, organized resource, ensuring your organization’s expert knowledge doesn't disappear when a long-term employee leaves.

3. **Documentation Workflows**: I set up reliable systems that make writing and updating instructions a natural part of the software development lifecycle ensuring your help guides never fall behind your latest software updates.

4. **Training resources**: I create training manuals for large organizations adopting new software, so your staff spends less time feeling stuck and more time getting their actual work done correctly.

## About this portfolio

This portfolio is built following a Docs-as-Code workflow to mimic modern engineering environments.

Static Site Generator: Built using Hugo for fast compilation, clean content architecture, and local development flexibility.

Content & Formatting: Written entirely in Markdown, utilizing custom Markdown Render Hooks in Go templates to dynamically manage link behaviors and HTML attributes.

Version Control: Managed via Git and hosted on Netlify, allowing for seamless tracking of documentation updates and deployment pipelines.

Theme is by [hanwenguo/hugo-theme-nostyleplease](https://github.com/hanwenguo/hugo-theme-nostyleplease?tab=readme-ov-file)
